**Auth**: All protected routes require `Authorization: Bearer <token>`  
**Content-Type**: `application/json` (unless specifying `multipart/form-data`)

## Auth

### POST /auth/signup

Registers a new **Admin** user, creates their **Company** (currency auto-detected via restcountries.com based on country code), and generates a session JWT.

> **No auth required**

**Request Body**

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `email` | string | ✅ | Valid email address |
| `password` | string | ✅ | Min 8 characters |
| `full_name` | string | ✅ | User's display name |
| `company_name` | string | ✅ | Name of the company to create |
| `country_code` | string | ✅ | ISO 3166-1 alpha-2 (e.g. `IN`, `US`, `GB`) |

```http
POST /api/v1/auth/signup
Content-Type: application/json

{
  "email": "admin@acme.com",
  "password": "SecurePass123",
  "full_name": "John Smith",
  "company_name": "Acme Corp",
  "country_code": "IN"
}
```

**Response `201`**
```json
{
  "message": "Signup successful.",
  "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
  "user": {
    "id": "mongo-object-id",
    "company_id": "mongo-company-id",
    "email": "admin@acme.com",
    "full_name": "John Smith",
    "role": "Admin",
    "createdAt": "2026-03-29T04:45:00.000Z"
  },
  "company": {
    "id": "mongo-company-id",
    "name": "Acme Corp",
    "default_currency": "INR",
    "country_code": "IN",
    "createdAt": "2026-03-29T04:45:00.000Z"
  }
}
```

**Errors**

| Status | Cause |
|--------|-------|
| `400` | Email already registered |
| `422` | Zod validation failure (missing fields, invalid country code) |
| `500` | DB error during company/user creation |

```http
  "full_name": "Jane Doe",
  "role": "Manager"
```

```json
{
```

## Expenses

### POST /expenses/manual

Allows an Employee to manually submit an expense directly (with an optional receipt file upload) without going through the OCR draft phase. Directly triggers the Approval Engine.

> **Auth required** — any role

**Request format:** `multipart/form-data`

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `amount` | number | ✅ | Expense amount |
| `currency` | string | ❌ | ISO 4217 (defaults to USD) |
| `title` | string | ❌ | Expense title |
| `description` | string | ❌ | Description |
| `category` | string | ❌ | Category |
| `receipt` | File | ❌ | PNG, JPG, or PDF (max 5MB) |

```http
POST /api/v1/expenses/manual
Authorization: Bearer <token>
Content-Type: multipart/form-data; boundary=---Boundary...

---Boundary...
Content-Disposition: form-data; name="amount"

12500
---Boundary...
Content-Disposition: form-data; name="title"

Flight to Mumbai
---Boundary...
Content-Disposition: form-data; name="receipt"; filename="ticket.pdf"
Content-Type: application/pdf

<file payload>
---Boundary...--
```

**Response `201`**
```json
{
  "data": {
    "id": "mongo-expense-id",
    "title": "Flight to Mumbai",
    "status": "Pending_Approval",
    "receipt_url": "/uploads/receipts/1723223030-ticket.pdf"
  }
}
```

### POST /approvals/:id/decide

Submit an **Approve** or **Reject** decision on a pending approval request.

> **Auth required** — `Manager` or `Admin` (must be the assigned approver)

> ⚠️ **Rejection requires a `comment`** — returns `400` if omitted.

**URL Params**

| Param | Description |
|-------|-------------|
| `id` | Mongo ObjectID of the `approval_request` |

**Request Body**

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `decision` | `"Approved"` \| `"Rejected"` | ✅ | The verdict |
| `comment` | string | ✅ if Rejected | Reason for rejection |

```http
POST /api/v1/approvals/mongo-object-id/decide
Authorization: Bearer <manager_token>
Content-Type: application/json

{
  "decision": "Approved",
  "comment": "Looks good."
}
```

**Rejection example**
```json
{
  "decision": "Rejected",
  "comment": "Receipt is unclear, please resubmit."
}
```

**Response `200` — outcomes**

| `outcome` | Meaning |
|-----------|---------|
| `"Approved"` | All steps complete — expense fully approved |
| `"Advanced"` | Step complete — moved to next approval step |
| `"Pending"` | Percentage rule — waiting for more approvers |
| `"Rejected"` | Expense rejected, all other requests skipped |

```json
{ "outcome": "Advanced", "next_step": 2, "expense_id": "mongo-expense-id" }
```

**Errors**

| Status | Cause |
|--------|-------|
| `400` | Request already decided / rejection missing comment |
| `403` | You are not the assigned approver |
| `404` | Request not found |

### POST /approvals/rules

Creates a new approval rule for the company's workflow.

> **Auth required** — `Admin` only

**Request Body**

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `name` | string | ✅ | Display name for the rule |
| `step_order` | integer | ✅ | Execution order (1 = first) |
| `rule_type` | `"Sequential"` \| `"Percentage"` \| `"Role_Override"` \| `"Direct_Manager"` | ✅ | Rule behaviour |
| `min_amount` | number | ❌ | Only trigger when expense ≥ this |
| `max_amount` | number | ❌ | Only trigger when expense ≤ this |
| `category` | string | ❌ | Only trigger for this category |
| `config` | object | ✅ | Rule-specific JSON config (see below) |

**`config` examples by rule type**

```json
// Direct_Manager — routes directly to the submitting employee's assigned manager_id
{ "approver_role": "Direct_Manager" }

// Sequential — all specific-role users are assigned, one needs to approve
{ "approver_role": "Manager" }

// Percentage — X% of all users with a role must approve
{ "approver_role": "Manager", "percentage": 60 }

// Role_Override — any user with this role can approve unilaterally
{ "override_role": "Admin" }
```

```http
POST /api/v1/approvals/rules
Authorization: Bearer <admin_token>
Content-Type: application/json

{
  "name": "Line Manager Review",
  "step_order": 1,
  "rule_type": "Direct_Manager",
  "min_amount": null,
  "max_amount": null,
  "category": null,
  "config": { "approver_role": "Direct_Manager" }
}
```

**Response `201`**
```json
{
  "data": {
    "id": "mongo-object-id",
    "company_id": "mongo-company-id",
    "name": "Line Manager Review",
    "step_order": 1,
    "rule_type": "Direct_Manager",
    "config": { "approver_role": "Direct_Manager" }
  }
}
```
